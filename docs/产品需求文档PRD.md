# 智租SaaS - 长租公寓管理系统
## 产品需求文档 v1.0

---

## 一、产品概述

### 1.1 产品定位
面向长租公寓房东的SaaS管理系统，支持多房东数据隔离，提供房源管理、合同管理、智能设备管控、费用管理、数据看板等核心功能。

### 1.2 目标用户
- **房东**：管理名下房源、合同、设备、收租
- **房客**：查看租房信息、签合同、缴水电费、使用智能锁
- **总控管理员**：平台运营方，管理所有房东和全局数据

### 1.3 核心价值
- 一个APP，房东/房客/管理员三端合一
- 智能设备实时监控，降低运营成本
- 多房东数据严格隔离，保障隐私安全

---

## 二、用户角色与权限矩阵

| 功能模块 | 房东 | 房客 | 管理员 |
|---------|------|------|--------|
| 房源管理 | 仅自己名下 | 查看已租房源 | 全部房源 |
| 合同管理 | 创建/归档/到期提醒 | 签署/查看 | 全部 |
| 智能设备 | 监控/充值/改密码 | 充值/查看记录 | 全部 |
| 费用管理 | 账单生成/收缴 | 在线缴费 | 全部 |
| 数据看板 | 个人数据 | 无 | 平台总览 |
| 商城购物 | 房东价 | 租客房 | 管理员配置 |
| 房东管理 | 无 | 无 | 增删改查 |
| 全局统计 | 无 | 无 | 全部 |

## 三、功能模块详细需求

### 3.1 登录与身份识别

**登录流程：**
1. 用户输入手机号 + 验证码
2. 后端根据手机号匹配用户角色（房东/房客/管理员）
3. 返回 JWT Token + 角色标识 + 用户信息
4. Flutter端根据角色跳转对应首页

**角色判定规则（Mock阶段）：**
- 手机号尾号 0-3：房东
- 手机号尾号 4-7：房客
- 手机号尾号 8-9：管理员

### 3.2 房东端 - 首页仪表盘

**顶部区域：**
- 搜索框：提示文字"搜房源、合同、设备"，点击弹出AI智能搜索（支持提示词搜索房源/合同/设备）
- 消息图标：显示未读消息数量
- Logo标识

**快捷入口（第一行4个）：**
- 创建合同：跳转合同模板选择
- 我的账单：弹出账单明细/合并账单
- 空房分享：弹出分享筛选（类型/状态/小区）
- 用电分析：跳转经营分析页

**功能入口（第二行，网格布局）：**
房源管理 | 我的合同 | 我的钱包 | 费用管理
我的设备 | 收房管理 | 数据统计 | 预定管理
我的入住 | 更多（增值服务/充电桩/用户地图等）

**新手指引栏：**
蓝色标签"新手指引" + 文字"点击进入智慧房东使用教程" + 箭头

**数据看板（核心数据）：**
- 近30日合同到期：X户（橙色预警）
- 合同逾期数量：X户（红色预警）
- 近30日待收金额：X万元（绿色）
- 其中逾期：X万元（红色）
- 全部房源/已租/空置（数字+趋势）
- 出租率/续签率/收房均价

**异常提醒区：**
电表不足5度 | 电表断电 | 水表不足1吨 | 门锁电量不足 | 门锁冻结

**待办工单：**
退租审批(X) | 账单审批(X) | 合同审批(X) | 在线维修(X)

### 3.3 房东端 - 房源管理

**顶部操作栏：**
- 房源类型下拉切换：整租/合租/独栋
- 帮助图标
- 新增按钮（+号）

**搜索框：**
提示文字"小区名/房源名/租客姓名/手机号"

**筛选栏：**
- 小区筛选：陆家嘴花园/浦东公寓/前滩小区/静安公寓/全部
- 状态筛选：未出租/已出租/签订中/预定中
- 更多筛选：户型/面积/租金范围
- 排序：默认排序/价格升序/价格降序

**标签栏（横向滚动）：**
全部 | 未出租 | 已出租 | 签订中 | 房租已逾期 | 合同快到期

**房源统计区：**
- 整租总房源数统计
- "关于房态图"入口

**房源列表：**
- 房源卡片：类型标签 + 状态标签 + 月租金
- 房源名称 + 地址
- 租客信息（如有）
- 点击跳转详情

### 3.4 房东端 - 设备管理

**顶部切换：**
- 设备列表（默认）
- 公共设备

**搜索框：**
提示文字"请输入房源名/小区名"

**分类标签（选中态蓝色下划线）：**
电表(X) | 水表(X) | 门锁(X)

**筛选栏：**
- 电表：少于X度/离线时/断电/在线
- 水表：少于X吨/离线时/断电/在线
- 门锁：预付费/后付费/在线/离线

**设备列表：**
- 设备卡片：设备图标 + 设备名 + 关联房源

**底部：**

**功能列表（图标+文字）：**

### 3.6 房东端 - 合同管理

**合同列表页：**
- 标签栏：全部/待签署/生效中/已到期

## 四、非功能性需求

### 4.1 性能要求
- 页面加载时间 < 2秒
- 接口响应时间 < 500ms
- 支持1000+房源流畅列表滚动

### 4.2 安全要求
- 所有接口需要JWT认证
- 房东数据严格隔离（同租户隔离）
- 密码加密存储（BCrypt）
- HTTPS全链路加密

### 4.3 兼容性
- 支持iOS 12+ 和 Android 8+
- 微信小程序同步适配（预留）
- 企业微信/钉钉消息推送预留
