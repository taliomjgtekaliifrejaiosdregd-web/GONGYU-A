## 八、索引设计

```sql
-- 用户手机号（登录查询）
CREATE INDEX idx_users_phone ON users(phone);

-- 房源按房东查询
CREATE INDEX idx_rooms_landlord ON rooms(landlord_id);
CREATE INDEX idx_rooms_status ON rooms(status);
CREATE INDEX idx_rooms_type ON rooms(type);

-- 合同按房源查询
CREATE INDEX idx_contracts_room ON contracts(room_id);
CREATE INDEX idx_contracts_status ON contracts(status);
CREATE INDEX idx_contracts_end_date ON contracts(end_date);

-- 账单按房东查询
CREATE INDEX idx_bills_landlord ON bills(landlord_id);
CREATE INDEX idx_bills_tenant ON bills(tenant_id);
CREATE INDEX idx_bills_status ON bills(status);
CREATE INDEX idx_bills_due_date ON bills(due_date);

-- 设备按房东查询
CREATE INDEX idx_devices_landlord ON devices(landlord_id);
CREATE INDEX idx_devices_type ON devices(type);
CREATE INDEX idx_devices_status ON devices(status);
```

## 九、多租户数据隔离策略

所有业务表都包含 `landlord_id` 字段，查询时强制带上 landlord_id 条件：

```sql
-- 正确：带租户隔离
SELECT * FROM rooms WHERE landlord_id = ? AND status = 'available';

-- 错误：缺少租户隔离（禁止执行）
SELECT * FROM rooms WHERE status = 'available';
```

前端每次请求必须携带 JWT Token，后端中间件解析后注入 landlord_id 到请求上下文。
