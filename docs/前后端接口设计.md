# 智租SaaS - 前后端接口设计 v1.0

## 一、接口规范

### 基础规范
- **Base URL**: `https://api.zhizu.com/v1`
- **认证方式**: JWT Bearer Token
- **Content-Type**: `application/json`
- **字符编码**: UTF-8

### 统一响应格式
```json
{
  "code": 0,           // 0=成功，其他=错误码
  "message": "success", // 提示信息
  "data": {},          // 业务数据
  "timestamp": 1700000000
}
```

### 错误码
| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 2001 | 未登录 |
| 2002 | Token过期 |
| 3001 | 权限不足 |
| 4001 | 资源不存在 |
| 5001 | 服务器错误 |

## 八、小程序同步适配注意事项

### 8.1 统一接口设计原则
- 所有接口返回标准JSON，不使用Flutter特有数据类型
- 日期格式统一为 ISO 8601（YYYY-MM-DD 或 YYYY-MM-DD HH:mm:ss）
- 金额统一用 DECIMAL(10,2)，单位为元
- 分页统一用 page + page_size + total
- 文件上传统一用 base64 或 OSS URL

### 8.2 Flutter端特殊处理
- DateTime: 使用 `DateTime.parse()` 解析，存储为 UTC
- 图片缓存: 使用 cached_network_image 插件
- 状态管理: 使用 Provider 或 Riverpod（小程序端用 wx.request 同步）

### 8.3 小程序适配点
- 登录：微信授权登录（替换手机号验证码）
- 支付：微信支付 wx.requestPayment
- 分享：微信分享 wx.showShareMenu
- 定位：高德地图 wx.getLocation
- 推送：微信订阅消息 wx.requestSubscribeMessage
- 扫码：wx.scanCode（设备绑定扫码）
- 文件：wx.chooseMessageFile（身份证上传）
- 设备蓝牙：wx.openBluetoothAdapter（智能锁蓝牙直连，需原生开发）

### 8.4 需要原生开发的功能
- 智能锁蓝牙BLE通信（Flutter无法直接操作蓝牙）
- 身份证OCR识别（腾讯云AI）
- 活体检测（支付宝/微信实名认证）
- 视频签署（WebRTC）
